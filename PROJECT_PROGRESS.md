## 项目概览

- 项目类型：Vue 3 + Vite 前端项目。
- UI/状态/请求栈：Element Plus、Tailwind CSS、Pinia、Vue Router、Axios。
- 当前主要业务方向：CargoSoon 物流服务入口、AI 咨询/聊天、运价推荐、货物追踪、仓储与 1688 sourcing 入口。

## 已完成页面

- `/` 首页：当前指向 `src/views/home2/index.vue`，包含 CargoSoon 品牌首屏、AI 搜索输入、热门搜索、快捷服务入口和信任信息区。
- `/home2` 首页备用入口：同样加载 `src/views/home2/index.vue`。
- `/chat` 聊天页面：包含全屏聊天面板和右侧推荐价格列表，发送消息后会刷新推荐价格。
- `/about` 关于页：已有路由和基础页面，但仍是默认占位内容。
- `/:pathMatch(.*)*` 404 页面：已有基础未找到页面和返回首页链接。

## 已接入的功能模块

- 全局布局：`src/layout/index.vue` 根据首页和非首页切换不同导航样式。
- 浮动聊天：`src/App.vue` 在非 `/chat` 页面展示 WhatsApp 浮动按钮和聊天组件。
- 首页 AI 入口：`src/views/home2/index.vue` 可将用户输入发送到全局聊天组件。
- 热门搜索：点击热门搜索词可直接发送到聊天组件。
- 附件/图片选择入口：首页已有文件选择交互，占位到输入框文本。
- 语音输入入口：支持浏览器 SpeechRecognition 时可填入识别结果。
- 老版工作台首页：`src/views/home/index.vue` 已存在物流看板、运价、推荐商品、追踪弹窗等模块，但当前未在路由中作为默认首页使用。
- API 模块：已存在 tracking、warehouse、user、dropshipping 等接口封装。

## 未完成功能

- `/about` 仍是模板占位页，需要替换为 CargoSoon 真实介绍内容。
- 首页快捷入口中的 `/main/pricelist`、`/order/tracking`、`/warehouse/SKUManagement`、`/new/chat` 等路径未在当前本地路由表中注册，需要确认是外部系统路径还是本项目后续页面。
- 首页附件上传、图片搜索目前只把文件名写入输入框，尚未接入真实上传、识别或搜索流程。
- 首页“Refresh”按钮当前打开运价页，没有真正刷新热门搜索词。
- `/chat` 页面中的 booking 操作跳转到 `shippingList` 命名路由，但当前路由表未定义该路由。
- README 仍是 Vite 默认说明，需要补充项目启动、环境、接口、部署和常见问题。

## 当前问题

- `src/layout/index.vue` 模板中调用了 `goSignUp`，但脚本中没有定义该方法，点击 Sign up 可能报错。
- `src/api/axios.js` 中出现两个 `baseURL` 配置，后者会覆盖前者；当前请求固定指向 `https://app.cargosoon.com/api`，本地和测试环境切换不够清晰。
- `src/api/tracking.js`、`src/api/warehouse.js`、`src/api/user.js` 中部分中文注释显示为乱码，需要统一文件编码或重写注释。
- 首页和布局中存在多个未注册的内部路径，可能导致本地开发环境跳转到 404 或外部系统不可用。
- 项目尚未看到自动化测试或页面级验收脚本，后续改动需要手动验证主要路由。
